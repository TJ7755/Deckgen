Process

Five phases, one brief.

01
Evidence

Web search and workspace file scan surface domain context before any writing begins.

02
Outline

The LLM proposes a story structure. Review, revise with plain-English instructions, or approve.

03
Design

A unique design system is generated per deck — font pair, colour palette, one signature visual move.

04
Content

Each slide is written in full: captions, statistics, quotes, and chart data drawn from the evidence gathered.

05
Build

reveal.js is cloned, images resolved via Wikimedia Commons, and a self-contained HTML file written to disc.

Palette

Ten slide types.

Title Card Opens the deck; brief and subtitle set the frame.
Transition Sharp narrative pivot between major story beats.
Image Hero Full-bleed photograph with a short anchor label.
Caption Card Image with contextual annotation alongside.
Chart — Bar Vertical bar chart generated from real or modelled data.
Chart — Line Time-series line chart with branded axis styling.
Data Table Structured data with header row and value columns.
Quote Callout Attributed pull quote from a named source.
Stat Callout Single large statistic with a supporting context line.
Comparison Side-by-side contrast of two states or entities.

Installation

Quick start.

1 — Clone, install, and register the command globally

git clone https://github.com/TJ7755/Deckgen.git
cd deckgen
npm install
npm install -g .

2 — Set your API key

echo "GEMINI_API_KEY=your_key_here" > .env

3 — Run from any directory

deckgen

Pass a brief directly — skip the prompts

deckgen --brief "The future of urban cycling" \
        --provider gemini \
        --depth focused \
        --variant dark \
        --serve

Check environment and credentials

deckgen doctor
